CMS Case Hierarchy

To understand GetCase and Subsequent Filing, you need to first understand how the Journal Tech case management system (CMS) tracks civil cases. For example, civil cases can be a:

  • simple dispute between two parties based on a single complaint filed by the plaintiff
  • contract dispute involving the same litigants in a complaint and cross-complaint
  • construction defect case involving MANY litigants with different attorneys across multiple cross and counter complaints
  • etc

This range requires a data structure that (i) manages litigants playing different roles across multiple complaints and (ii) allows the Court to precisely calculate first appearance fees, track judgments by complaint, generate statewide statistical reports, etc.

For this reason, we have a custom NIEM extension called Complaint (or "Sub-Case")which is a sub-entity of Case and tracks party information based on the complaint filed. For example:
Complaint (“original”)
Plaintiff: B. Schmidt
      Attorney - D. Johnson
Defendant: K. Long
      Attorney - C. Templeton

Complaint 2 (“counter-claim”)
Plaintiff: K. Long
      Attorney - C. Templeton
Defendant: B. Schmidt
      Attorney - D. Johnson

Complaint 3 (“cross-complaint”)
Plaintiff: B. Schmidt
      Attorney - L. Davis
Defendant: L. Williamson
      Attorney - K. Jordan
      Attorney - P. Grossman

In this model, each complaint has it’s own ID and each party (not person) has their own ID. So when the end user is asked to identify which parties a Proof of Service is filed against, they must select a party from a particular Complaint.
