Get Recording Status

The GetRecordingStatus API endpoint is a custom extension to FilingReview designed to:
  1. Replace the weekly Vendor Transaction Report (PDF) emailed to EFSPs.
  2. Give a more accurate picture of a filing by returning both Filing Status and Recording Status
  3. Provide EFSPs a way to run reports at their convenience.

Get Recording Status (by IdentificationID)

This sample demonstrates how to request your report by Identification ID. RecordingStatus is driven by credentials and will only return filings submitted by your EFSP.

Sample Request   |   Sample Response

Request Parameter Description
IdentificationID Required. The transaction identifier based on your value in <IdentificationID>. RecordingStatus is driven by credentials and will only return filings submitted by your EFSP.
IdentificationCategory Required. Set to efm if you are passing the EFM Transaction Id as the <IdentificationID>. Set to efsp if you are passing your internal transaction identifier (sent during ReviewFiling) as the <IdentificationID>.

Request Parameter Description
StartDate Required. The starting date of the date range.
EndDate Required. The ending date of the date range. Currently the max date range is 7 days. We may extend this date range if the performance impact is reasonably small.

Results

RecordingStatus is driven by credentials and will only return filings submitted by your EFSP.

Element Description
FilingStatusCode Signifies the overall filing status as determined by the Court.
  • RECEIVED_UNDER_REVIEW = Filing was received and is under review by the court
  • ACCEPTED = Filing was accepted
  • PARTIALLY_ACCEPTED = Filing was received, some documents were accepted, and some documents were rejected
  • REJECTED = Filing was received and completely rejected
RecordingStatus Signifies the overall filing status as determined by the Court.
  • RECEIVED_UNDER_REVIEW = Filing was received and is under review by the court
  • COMPLETE = Filing was accepted by the Court and all further processing is complete.
  • REJECTED = Filing was rejected by the Court so no further processing has occurred.
  • PROCESSING = The filing was accepted by the Court but further processing is occurring. This can mean anything ranging from (i) additional documents need to be processed by the Court, (ii) the filing requires the Court to fix data before being submitted into the CMS, (iii) there may be a technical issue preventing the full lifecycle to be completed, etc. In future version, we hope to have "codes" that better articulate the exact issue preventing the recording status from showing complete.
