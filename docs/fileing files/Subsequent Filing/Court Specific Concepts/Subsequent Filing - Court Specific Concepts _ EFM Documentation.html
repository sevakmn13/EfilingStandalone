Alameda Superior Court

With Hearing Boolean for APP045

A required Boolean field for the document "APP045 - Application for Determination of Good Faith Settlement" which will trigger a fee change based on whether the document is filed with or without a hearing. If true the fee will be $60.00, if False it will be reduced to $20.00.

The name of the new field is IS_WITH_HEARING and is a required field for this document. It has to be submitted with either a Y or N value.

 
<ns9:DocumentFilingMetaData>
  <ns9:documentFilingMetaDataItem>
    <ns9:docValueMetaDataItem>
     <ns8:code>IS_WITH_HEARING</ns8:code>
     <ns8:classType>codeList</ns8:classType>
     <ns8:valueRestriction>new-data</ns8:valueRestriction>
  </ns9:docValueMetaDataItem>
  <ns9:codeValue>Y</ns9:codeValue>
 </ns9:documentFilingMetaDataItem>
<ns9:documentFilingMetaDataItem>

Los Angeles Superior Court

Proposed Orders

Background

The JTI EFM stores every incoming XML envelope and records unique transaction IDs at the package and document level

Unique EFM System Generated:

<ns3:IdentificationID>a37aa4d5b386</ns3:IdentificationID>

EFSP Generated Document Control ID: 

<ns3:DocumentFileControlID>754</ns3:DocumentFileControlID>

These 2 unique identifiers will be the basis for sending back an order when its filing status has changed from RECEIVED to FILED by the court.

Proposed Order Workflow – Court CMS Level

  1. In the court CMS, the Proposed Order is routed internally for further processing by the Judicial Officer, Research Attorney, or Operations Clerk
  2. At some point in the future, the Proposed Order is acted upon and FILED.
  3. Filing the order automatically triggers the process of electronically returning the FILED Order back to the originating EFSP.

Proposed Order Workflow – EFSP Filed Order NFRC

The return NFRC will include the Identification ID and Document Control ID from the originating XML package that sent in the Proposed Order.

Deferred Filing Fees (DRAFT)

NOTE: Los Angeles Superior Court is reviewing their back office financial workflow to support this feature. As a result this feature is on hold. The documentation below provides a general idea of the what is coming.

Los Angeles Superior Court gives end-users the option to defer payment of court filing fees on certain filings. This initially applies to "Appeal - Notice of Appeal/Cross Appeal Filed" but could be extended to others in the future.

IMPORTANT - Vendors are *NOT RESPONSIBLE* for collecting deferred filing fee payments. These deferred fees must be paid in person based on the deferred payment rules defined by the Court (see "Required Text" below).

Implementation

The implementation of deferred payments uses our standard Document Metadata model. We've created a new metadata code called DEFERRED_PAYMENT to support this feature. Here's an example metadata item definition for "Appeal - Notice of Appeal/Cross Appeal Filed":

<ns6:metadataItems>
    <ns6:code>DEFERRED_PAYMENT</ns6:code>
    <ns6:name> PAYMENT OPTIONS </ns6:name>
    <ns6:description>FEES CANNOT BE SUBMITTED VIA EFILING AFTER NOTICE OF APPEAL HAS BEEN ACCEPTED FOR EFILING. Fees must be paid within 10 days to the Appeals unit of Stanley Mosk Courthouse. Court of Appeal fees may also be paid directly to the Court of Appeal.</ns6:description>
    <ns6:classType>codeList</ns6:classType>
    <ns6:codeList>DEFERRED_PAYMENT</ns6:codeList>
</ns6:metadataItems>

As you can see above, the classType is a standard codelist which references DEFERRED_PAYMENT codelist found in Court Policy.

Required Text

Los Angeles Superior Court requires that the text is defined within the <description> tag be displayed to the end-user!!.

It's up to each vendor how to implement the display of the required text. You can either (i) always show the text or (ii) show only when user selects YES value. For example, Journal Technologies displays the description only if the "YES" value is selected by the end-user:

Fee Calculations

If the end-user opts for deferred payment, court filing fees will be deferred on ReviewFiling but both the (i) Court Technology Fee and (ii) JTI Transaction Fee will be assessed as normal. For example:

  • Court Filing Fees: $0
  • Court Technology Fee: $2.25
  • JTI Transaction Fee: $1.95

EFSPs may need to perform multiple GetFeesCalculation calls to preview the total filing fees.

  • Passing metadata item DEFERRED_PAYMENT value of "NO" will return the normal filing fee amount.
  • Passing metadata item DEFERRED_PAYMENT value of "YES" will return $0.

Pagination

Pagination is a new tool built into the GetCase API. This is not required but is recommended when working with large cases, to limit the number of parties pulled by page size, which reduces load times and file sizes.

Offset declares which dataset you will receive. The offset will increase based on the page size number. Page two with page size of 10 would be an offset of 10. Page 3 would be 20. 

<ns5:pageSize>10</ns5:pageSize>
<ns5:PartyCriteria>
   <ns5:offset>0</ns5:offset>
   <ns5:metadataFilter>CASE_PARTICIPANT_ATTORNEY</ns5:metadataFilter>
</ns5:PartyCriteria>

With this query, you are able to filter out specific party types such as Plaintiffs, defendants, or attorney with four Lookup List items.
Filters which are allowed:

  • CASE_PARTICIPANT_PLAINTIFF
  • CASE_PARTICIPANT_DEFENDANT
  • CASE_PARTICIPANT_ATTORNEY
  • CASE_PARTICIPANT_ALL_PARTIES_EXCEPT_ATTORNEY

Placer Superior Court

Consent to eService

Per California Rule 2.251 , a party or other person may indicate consent to accept electronic service and distribution of documents. This may include documents generated from the Court such as Judgments and Orders.

The rule DOES NOT prescribe specific language for a provision of a term of service when the filer consents to electronic service, but does require that any such provision be clear. Consent to Electronic Service and Notice of Electronic Service Address (form EFS-005-CV) provides an example of language for consenting to electronic service.

The JTI EFSP system currently displays the following language:

By checking this box, I consent to electronic service of notices and documents related to this case. PLEASE NOTE: The email address provided to the court at the time of filing will be used for electronic service and distribution of documents.

Subsequent Filing
For subsequent filing, the consent to eService is a standard document metadata flag. Please review the following scenarios in the Placer County Review Filing Examples :

  • First paper filing without representation - Consent to eService
  • First paper filng with new representation - Consent to eService
  • Motion filing by attorney with eService consent
