Subsequent Filing

The document attributes determine if the document is allowed for eFiling based on the Form Group, Case Types and Case Categories. The document may be allowed for subsequent filings but not case initiation.

In order to differentiate subsequent filings from case initiation, your document list API call must include subFiling=true. You will be able to see documents that are available for case categories that were not specified on the document definitions otherwise. Please refer to this example: 

/ws/rest/ecourt/niem/documentList?caseType=CL&subFiling=true

Subsequent Fee Waivers

Fee Waivers can be filed on subsequent fee documents. (e.g. Answers, Cross- Complaints). Fee Waiver tags are passed in the FeesCalculationQueryMessage for subsequent filings. The structure for Fee Waivers in subsequent XML differs from case initiation. Please refer to this example: 

<ns9:additionalInfoTags>
	<ns9:tagType>FEE_EXEMPTION</ns9:tagType>
	<ns9:tagValue>FEE_WAIVER</ns9:tagValue>
</ns9:additionalInfoTags>

Erroneously Sued As

Erroneously Sued As can be added to a subsequent document as follows: 

<ns9:additionalInfoTags>
	<ns9:tagType>ERRONEOUSLY_SUED_TRUENAME</ns9:tagType>
	<ns9:tagValue>John Paul Taylor Jr</ns9:tagValue>
</ns9:additionalInfoTags>

Name Extension

Name Extension allows the Document Caption to be extended by the submitter. e.g Declaration (name extension); user will be presented with a text box to add [text input] (re: Order to Compel Accounting). CMS will record the filing as Declaration re: Order to Compel Accounting

In the document metadata, the Form Group "EFNE" determines if the document allows a name extension to be used. If so, a text box is required for user to input the additional text. Please refer to this example: 

<ns10:nameExtension>User adds extension here</ns10:nameExtension>

Unlawful Detainer

Case Search & Access

Per Rules of Court, Unlawful Detainer cases are deemed confidential and locked from public view at case initiation.

Public Disclaimer

JTI EFM requires all EFSPs to alert users of their electronic portal/gateway with the following disclaimer when searching for Unlawful Detainers:

Code of Civil Procedure §1161.2 (a) limits access to unlawful detainer cases. Accessing an unlawful detainer case through this system could provide confidential information regarding the case. By accessing this case, you agree not to disclose, copy, publish, sell, or otherwise use confidential case information you access for any other purpose. Doing so may expose you to legal liability or result in criminal consequences.

Public Access Question

EFSPs are required to prompt all users, under penalty of perjury, to answer the following question:

Are You a Party to this Case?

Access Tracking and Data Capture

The end user’s response is required to be captured. If the user states they are not a party to the case, they should not able to proceed further. If answered in the affirmative, then the user proceeds to accessing the Unlawful Detainer case. At this point the EFSP is required to capture the answer to the question, the user name and Case Number involved in the search.

Criminal / Juvenile Case Types

The following case type groups are being returned at the case level (CivilCaseExt) rather than within a subcase (Complaint):
  • Juvenile cases – where CaseTypeText begins with "3".
  • Criminal cases – where CaseTypeText begins with "9".

These case types will appear directly under: 

CaseResponseMessage → CivilCaseExt

or 

NotifyFilingReviewCompleteRequestMessage → ReviewFilingCallbackMessageExt → CivilCaseExt

For Juvenile cases (3-code), sensitive party information is masked before the data is sent to external systems:
  • CaseTitleText → "REDACTED"
  • PersonGivenName → "Confidential"
  • PersonMiddleName → "" (empty)
  • PersonSurName → "" (empty)
  • JudgmentName → "Confidential"
Criminal cases (9-code) are not redacted but will follow the same structural pattern at the case level.
