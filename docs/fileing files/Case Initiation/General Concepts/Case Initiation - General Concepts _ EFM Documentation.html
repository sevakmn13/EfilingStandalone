Case Type

Case Type is constructed using the SimpleValue found under the CASE_TYPE codelist in Court Policy. Each Court will likely have different Case Type codes so prepare your systems accordingly.

For example, here is the case type code for a Civil Unlimited filing in Los Angeles Superior Court: 

<ns6:CaseTypeText>CU</ns6:CaseTypeText>

Case Category

Case Category is constructed using the SimpleValue found under the CASE_CATEGORY codelist in Court Policy. Each Court will likely have different Case Category codes so prepare your systems accordingly.

For example, here is the case category code for a Contract: Insurance Coverage in Placer Superior Court: 

<ns3:CaseCategoryText>405300</ns3:CaseCategoryText>

Conditionally Sealed

New Case screens should display an optional checkbox (boolean) for a sealed case IF the case type selected is Civil Unlimited. The XML element during ReviewFiling is simply conditionallySealed. For example, in Los Angeles Superior Court: 

<ns1:Case xsi:type="ns6:CivilCaseTypeExt">
  <ns1:CaseCategoryText>1001</ns1:CaseCategoryText>
  <ns6:CaseTypeText>CU</ns6:CaseTypeText>
  <ns6:conditionallySealed>true</ns6:conditionallySealed>
</ns1:Case>

Jurisdictional Amount

For Civil litgation cases, the Jurisdictional Amount (or "Jurisdictional Grounds Code") is required and is used to determine appropriate filing fees for case initiation.

For example:
  • Under 10K (U10K)
  • Over 10K but under 25K (O10K)
  • 25K and Over (O25K)

<ns5:JurisdictionalGroundsCode>O10K</ns5:JurisdictionalGroundsCode>

NOTE: For Civil Unlimited cases, only 25K and Over will be accepted. It's suggested you only display 25K and Over to end users.

Demand Amount

This is an optional data element. Per operational discretion, the demand amount is not required on case initiating documents. 

<ns5:AmountInControversy xsi:type="ns4:AmountType">12000</ns5:AmountInControversy>

Participant Roles

Participant Roles should be filtered by case category. For complaints plaintiff and defendant roles will be used; petitioners and respondents for petitions; for parking appeals and administrative hearings appellant and respondent.
  • Plaintiff (PLAIN)
  • Appellant (APP)
  • Attorney (ATT)
  • Defendant (DEF)
  • Respondent (RES)

<p1:CaseParticipantRoleCode xmlns:p1="urn:oasis:names:tc:legalxml-courtfiling:schema:xsd:CommonTypes-4.0">PLAIN</p1:CaseParticipantRoleCode>

Entity Types

JTI only supports two entity types. There does not exist a Government Entity Type in the JTI Schema. All Government entities are of an EntityOrganization type. Suffix text can be added to any EntityPerson type. Suffixes are plain text and are not pre-defined suffix codes.

Organization Name Structure: 
<p3:OrganizationName>Tri-State Properties</p3:OrganizationName>

Person Name Structure with Suffix Text: 
<p2:PersonName> 
  <p2:PersonGivenName>Rodney</p2:PersonGivenName> 
  <p2:PersonSurName>Baker</p2:PersonSurName> 
  <p2:PersonNameSuffixText>Jr</p2:PersonNameSuffixText> 
</p2:PersonName>

  • Also Known As (AKA)
  • Alias (Alias)
  • Doing Business As (DBA)
  • Formerly Doing Business As (FDBA)
  • Formerly Known As (FKA)

Contact Information: Attorney & Self Represented

JTI does not require an extra data element that designates a party as self-represented. The EFM designates a party or parties is self-represented if there is no attorney associated in the XML. This applies for both Person and Organization Entity types. If attorney contact information is not provided then the party is deemed to self-represented and contact information, including email and mailing address, must be provided.

  • MAILING Address Type is required for all addresses
  • WORK Telephone Type is required for telephone number. Recommended format 9999999999 (no hyphens)
  • EMAIL Addresses are required for all attorneys and/or Self Represented Litigants and validated to standard local-part@domain,com format.

Contact name requirements are as follows:

Contact Name Requirement
First Name Required
Middle Name/Initial Optional
Last/Sur Name Required
Law Firm Name Required if No Attorney Name, otherwise optional
Attorney First Name Required
Attorney Middle Name Optional
Attorney Last Name Required

Document Requirement

Lead Documents
JTI has included the Form Group EFCI_LEAD indicator as part of the document list to identify FilingLeadDocument.

Snippet of DOCUMENT LIST – Court Policy File: 

<ns3:formGroups>
  <Row>
     <Value ColumnRef="code"> 
           <SimpleValue>EFCI_LEAD</SimpleValue>
      </Value>
  </Row>
</ns3:formGroups>

XML Snippet: 

<ns8:FilingLeadDocument st:id="doc0" xmlns:st="http://niem.gov/niem/structures/2.0" xsi:type="ns9:DocumentExtType">
  <ns1:DocumentBinary>
    <ns1:BinaryFormatStandardName xsi:type="ns4:TextType">application/pdf</ns1:BinaryFormatStandardName>
    <ns1:BinaryLocationURI>YOUR_URL_HERE</ns1:BinaryLocationURI>
  </ns1:DocumentBinary>
  <ns1:DocumentDescriptionText>COM040</ns1:DocumentDescriptionText>
  <ns1:DocumentFileControlID>29769</ns1:DocumentFileControlID>
  <ns1:DocumentIdentification>
    <ns1:IdentificationID>COM040</ns1:IdentificationID>
    <ns1:IdentificationSourceText>lasc</ns1:IdentificationSourceText>
  </ns1:DocumentIdentification>
  <ns1:DocumentSequenceID>0</ns1:DocumentSequenceID>
</ns8:FilingLeadDocument>

Complex Indicator

This Boolean allows filers to mark their new case as complex at the time of initial filing. The Complex indicator for case initiations is currently used in both LASC and Riverside.

The Court requires All EFSPs are to display the following notice to end-users when Complex is marked as True:

Note to filer: Per GC 70616(a), a $1,000.00 additional fee for case designated as complex (one fee for all plaintiffs) will be calculated with this transaction.

Screenshot of indicator:

If the answer is YES, your ReviewFiling must contain the following snippet: 

</ns6:CaseAugmentationExt>
<ns6:complexLitigation>true</ns6:complexLitigation>

More Than 12 Claims

For Small Claims cases, this Boolean allows filers to indicate they are a frequent filer with more than 12 claims at the time of initial filing. The "More than 12 claims filed in the last 12 months" checkbox for case initiations is currently used in courts like LASC, Alameda, Riverside and Sacramento.

Screenshot of indicator:

If the "More than 12 claims filed in the last 12 months" box is checked , your ReviewFiling must contain the following snippet: 

</ns6:CaseAugmentationExt>
<ns6:moreThan12Claims>true</ns6:moreThan12Claims>

For smaller courts like El Dorado, Madera and Solano, the "More than 12 claims" is indicated in the Jurisdictional Amount dropdown box as "Frequent Filer".

For example:
  • $1501 - $5000
  • Frequent Filer
  • Over $5K up to $12.5K
  • Up to $1500

<ns5:JurisdictionalGroundsCode>Frequent Filer</ns5:JurisdictionalGroundsCode>
