Case Initiation - How to Build a Payload

This document walks through the construction of a general ReviewFiling payload.

Download or view this XML sample. This sample will be used in the below discussion.

1. Request Information

<ns13:ReviewFilingRequestMessage>
  <ns2:SendingMDELocationID>
    <ns1:IdentificationID>YOUR_ENDPOINT_HERE</ns1:IdentificationID>
    <ns1:IdentificationSourceText>lasc</ns1:IdentificationSourceText>
  </ns2:SendingMDELocationID>
  <ns8:CoreFilingMessage xsi:type="ns9:CoreFilingMessageExtType">
    <ns1:DocumentFiledDate id="ref1">
      <ns1:DateTime>2019-12-17T11:04:49-08:00</ns1:DateTime>
    </ns1:DocumentFiledDate>
    <ns1:DocumentIdentification>
      <ns1:IdentificationID>YOUR_IDENTIFICATIONID_HERE</ns1:IdentificationID>
    </ns1:DocumentIdentification>
    <ns1:DocumentSubmitter>
      <ns2:EntityPerson id="Person1">
        <ns1:PersonName>
          <ns1:PersonFullName>YOUR_USERNAME_HERE</ns1:PersonFullName>
        </ns1:PersonName>
      </ns2:EntityPerson>
    </ns1:DocumentSubmitter>

Request Parameter Description
IdentificationID

(SendingMDELocationID)

 URL endpoint on your server where EFM will return all responses to for the lifecycle of this filing.
IdentificationID

(DocumentIdentification )

 This is your own unique filing identifier that is passed back during NotifyFilingReviewComplete. This identifier is mainly used by EFM for debugging purposes and auditing.
PersonFullName

(DocumentSubmitter)

 Optional. This parameter contains information about the end user who actually clicked "Submit". Usually this is just username. This might include email address in future. This is a data concept to associate the username with an audit event.

Request Parameter Description
CaseTypeText The case type code selected by the user. The case type value is defined in CaseTypeText under Court Policy.
CaseCategoryText The case category code selected by the user. The case category value is defined in CaseCategoryText under Court Policy. This value must be honor the relationship between Case Type and Case Category defined in Court Policy or the EFM will return a failure upon submission.
CaseParticipantExt These are your case participants including parties and attorneys. The role code value is defined in CaseParticipantRoleCode under Court Policy. The classType defined in document metadata drives the actual element structure for attorneys and parties. Notice the use of <EntityOrganization> and <EntityPerson> to capture information regarding organizations vs individuals.
JurisdictionalGroundsCode The Jurisdictional Grounds Code is selected by the user and used to determine appropriate filing fees for case initiation. The jurisdictional grounds value is defined in JurisdictionalGroundsCode under Court Policy.
AmountInControversy Sometimes called "jurisdictional amount", this is an optional data element. Per LASC operational discretion the demand amount is not required on case initiating documents. It denotes the amount at stake in the lawsuit. This value must be consistent and within the boundaries of the JurisdictionalGroundsCode.
relatedParticipants This critical section is how you relate (or map) parties, attorneys, documents, etc. The <associationType> values are defined in document metadata and describe the type of relationship. The REPRESENTEDBY element relates a <participant> with an attorney referenced by <relatedParticipant>. The FILEDBY and REFERS_TO elements relate a <participant> with a document <relatedDocument> .

All mappings are accomplished by first setting your own reference identifier in the id attribute of <CaseParticipantExt>. These are then referenced under <relatedParticipants> elements using the ref attribute.
CaseCourt This defines the court location where the filing is permitted. The <IdentificationSourceText> is a court location code that must be set by using our Court Location lookup. The <IdentificationID> will always be "Los Angeles Superior Court", the <IdentificationSourceText> will always be "lasc" and the <CourtName> will always be "Los Angeles Superior Court". Pass the zip code in the <LocationPostalCode> element

Request Parameter Description
FilingLeadDocument The lead document in the filing sequence. JTI EFM only accepts one lead document per filing.
FilingConnectedDocument Any additional documents attached to the filing
BinaryFormatStandardName The mime type of the document. Currently LASC only accepts application/pdf and mime types associated with MS word (ex. application/msword and application/vnd.openxmlformats-officedocument.wordprocessingml.document
BinaryLocationURI EFM will download the document from this location. It's the EFSPs responsibility to ensure the document is available.
DocumentDescriptionText The document code. This would likely be what "document" the end-user is filing. The document code value is defined in DocumentIdentification under Court Policy.
DocumentFileControlID Optional. Your own identifier for this document.
DocumentSequenceID The sequence number of this document in a filing.
messageToClerk Any additional message the end user wishes to communicate to the Court about the filing.
